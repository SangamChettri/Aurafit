# Fitness Platform Backend

Express.js backend API for the fitness platform.

## Setup

1. **Install dependencies:**
   ```bash
   npm install
   ```

2. **Configure environment:**
   - Copy `.env.example` to `.env`
   - Update database connection string
   - Set JWT secret

3. **Setup database:**
   ```bash
   npm run prisma:generate
   npm run prisma:migrate
   ```

4. **Initialize data:**
   ```bash
   npm run init
   ```
   This creates:
   - First admin user (admin@fitness.com / admin123)
   - Nepal locations
   - Sample exercises

5. **Start server:**
   ```bash
   npm run dev
   ```

## API Endpoints

### Authentication
- `POST /api/auth/register` - Register new user
- `POST /api/auth/login` - Login
- `POST /api/auth/logout` - Logout
- `GET /api/auth/me` - Get current user

### Users
- `GET /api/users/profile` - Get profile
- `PUT /api/users/profile` - Update profile
- `POST /api/users/profile/image` - Upload profile image

### Workouts
- `GET /api/workouts` - Get all workouts
- `GET /api/workouts/:id` - Get workout
- `POST /api/workouts` - Create workout
- `PUT /api/workouts/:id` - Update workout
- `DELETE /api/workouts/:id` - Delete workout
- `POST /api/workouts/:id/sets` - Add set
- `POST /api/workouts/:id/cardio` - Add cardio session

### Exercises
- `GET /api/exercises` - Get all exercises
- `GET /api/exercises/:id` - Get exercise
- `POST /api/exercises` - Create exercise (Admin/Coach)
- `PUT /api/exercises/:id` - Update exercise (Admin/Coach)
- `DELETE /api/exercises/:id` - Delete exercise (Admin)

### Progress
- `GET /api/progress` - Get progress entries
- `POST /api/progress` - Create progress entry
- `PUT /api/progress/:id` - Update progress entry
- `DELETE /api/progress/:id` - Delete progress entry
- `GET /api/progress/analytics/summary` - Get analytics (Premium)

### Goals
- `GET /api/goals` - Get all goals
- `POST /api/goals` - Create goal
- `PUT /api/goals/:id` - Update goal
- `DELETE /api/goals/:id` - Delete goal
- `PATCH /api/goals/:id/complete` - Mark goal complete

### Water Intake
- `GET /api/water` - Get water logs
- `GET /api/water/today` - Get today's intake
- `POST /api/water` - Log water intake
- `DELETE /api/water/:id` - Delete log

### Workout Plans
- `GET /api/workout-plans` - Get plans
- `GET /api/workout-plans/:id` - Get plan
- `POST /api/workout-plans` - Create plan
- `POST /api/workout-plans/generate` - Generate AI plan (Premium)

### Social
- `GET /api/social/connections` - Get friends
- `POST /api/social/connect/:userId` - Send friend request
- `PATCH /api/social/accept/:connectionId` - Accept request
- `GET /api/social/feed` - Get activity feed
- `POST /api/social/posts` - Create post
- `POST /api/social/posts/:postId/like` - Like post
- `POST /api/social/posts/:postId/comments` - Comment on post

### Coaching (Premium)
- `GET /api/coaching/coaches` - Get coaches
- `GET /api/coaching/assignments` - Get assignments
- `POST /api/coaching/assignments` - Request coach
- `GET /api/coaching/messages/:userId` - Get messages
- `POST /api/coaching/messages` - Send message

### Wearables
- `GET /api/wearables/syncs` - Get syncs
- `GET /api/wearables/last-sync` - Get last sync
- `POST /api/wearables/connect` - Connect device (Placeholder)
- `POST /api/wearables/sync` - Sync data (Placeholder)

### Subscription
- `GET /api/subscription/status` - Get status
- `POST /api/subscription/upgrade` - Upgrade (Placeholder)
- `POST /api/subscription/cancel` - Cancel

### Notifications
- `GET /api/notifications` - Get notifications
- `PATCH /api/notifications/:id/read` - Mark as read
- `PATCH /api/notifications/read-all` - Mark all as read

### Admin
- `GET /api/admin/dashboard` - Dashboard stats
- `GET /api/admin/users` - Get users
- `GET /api/admin/users/:id` - Get user details
- `PUT /api/admin/users/:id` - Update user
- `GET /api/admin/exercises` - Get exercises
- `GET /api/admin/logs` - Get admin logs

```env
ADMIN_EMAIL=admin@fitness.com

```

## Database

Uses Prisma ORM with MySQL. Schema is defined in `prisma/schema.prisma`.

## File Uploads

Uploads are stored in `public/uploads/`. Configured with Multer middleware.
